  5. Completar los espacios en blanco.
    • git add agrega archivos al próximo commit.
    • El comando git log muestra un registro con los commits de un repositorio.
    • git init crea un repositorio en el directorio de trabajo.
    • Para subir un repositorio local a uno remoto usamos git push.
    • Las 2 maneras de obtener un repositorio de Git son son o usando git init en un directorio, o clonando un repositorioexistente con git clone.
    • Si quiero hacer cambios en un repositorio compartido por un equipo de trabajo es conveniente hacerlos en una rama.
    • Cuando fusionamos 2 ramas puede haber un conflicto si el mismo archivo fue modificado en ambas ramas.
    • Para resolver un conflicto de merge hay que editar el archivo que genera el conflicto y marcarlo como resuelto con git add.
    • No podemos realizar un push a un remoto si nuestra copia local no está actualizada con los últimos cambios del repositorio remoto. En ese caso tendremos que realizar un pull primero antes de subir nuestro trabajo.

  6. Completar lo que falta en los siguientes comandos de Git.
    • git add . (agrega todos los archivos del directorio staging area).
    • git status -s (versión abreviada del estado del repositorio).
    • git log -10 (ver los últimos 10 commits).
    • git commit --amend (arregla el último commit).
    • git add index.html css (agrega index.html y el directorio css al index).
    • git checkout -b new branch (crea y cambia a una rama llamada new-branch).
    • git remote -v (muestra las URLs asociadas con cada remoto).
    • git pull (sincroniza la rama en la que estoy con la rama remota rastreada).
    • git merge hotflix (fusiona la rama hotflix con la rama en la que estoy parado).
